# AI Coding Rules & Standards for Video Generation POC

**Version**: 1.0  
**Last Updated**: 2026-05-07  
**Purpose**: Comprehensive guidelines for AI-assisted code generation to ensure consistency, quality, and production-readiness.

---

## 🎯 Core Principles

1. **Production-First**: Write production-grade code from the start, not POC-quality code
2. **Test-Driven**: Write tests before implementation (TDD)
3. **SOLID Principles**: Follow SOLID design principles religiously
4. **Type Safety**: Leverage TypeScript's type system fully - avoid `any`, use strict mode
5. **Explicit over Implicit**: Code should be self-documenting and obvious
6. **Fail Fast**: Validate early, throw meaningful errors
7. **Logging Everything**: Log all important operations with context
8. **Security First**: Never trust input, always validate and sanitize

## 🏗️ NestJS Architectural Patterns

### Module Structure
Every module MUST follow this structure:
```typescript
@Module({
  imports: [
    // External modules first
    ConfigModule,
    // Internal modules
    SharedModule,
  ],
  controllers: [DomainController],
  providers: [
    DomainService,
    // List all providers explicitly
  ],
  exports: [DomainService], // Export what others need
})
export class DomainModule {}
```

### Dependency Injection Rules
1. **Always use constructor injection** (never property injection)
2. **Inject interfaces, not implementations** when possible
3. **Use custom providers** for interface-based DI:
```typescript
{
  provide: 'IScriptGenerator',
  useClass: OpenAIScriptProvider,
}
```
4. **Scope**: Use `Scope.DEFAULT` (singleton) unless specific need
5. **Avoid circular dependencies** - restructure if needed

## 📝 Naming Conventions

### Classes
- **PascalCase** for all classes
- **Descriptive names**: `VideoGenerationService` not `VideoService`
- **Suffix with type**: `{Name}Service`, `{Name}Controller`, `{Name}Module`
- **Interface prefix**: `I{Name}` for interfaces (e.g., `IScriptGenerator`)

### Methods
- **camelCase** for all methods
- **Verb-first**: `generateVideo()`, `createJob()`, `validateInput()`
- **Boolean methods**: `is{Condition}`, `has{Feature}`, `can{Action}`
- **Async methods**: Always return `Promise<T>`, prefix with `async`

### Variables
- **camelCase** for variables
- **Descriptive names**: `videoJob` not `job`, `scriptContent` not `content`
- **Constants**: `UPPER_SNAKE_CASE` (e.g., `MAX_VIDEO_DURATION`)
- **Private properties**: prefix with `private readonly` when immutable

## 🔒 TypeScript Strict Rules

### Type Safety
1. **Never use `any`** - use `unknown` if type is truly unknown
2. **Enable strict mode** in tsconfig.json
3. **Explicit return types** on all public methods
4. **Avoid type assertions** (`as`) unless absolutely necessary
5. **Use generics** for reusable code

## ⚠️ Error Handling Patterns

### Custom Exception Hierarchy
```typescript
export class VideoGenerationException extends HttpException {
  constructor(message: string, cause?: Error) {
    super(
      {
        statusCode: HttpStatus.INTERNAL_SERVER_ERROR,
        message,
        error: 'VideoGenerationError',
        timestamp: new Date().toISOString(),
      },
      HttpStatus.INTERNAL_SERVER_ERROR,
    );
    this.cause = cause;
  }
}
```

### Error Handling Rules
1. **Always catch and re-throw** with context
2. **Log before throwing** (except in controllers)
3. **Use specific exceptions** not generic `Error`
4. **Include original error** as `cause`
5. **Never expose internal errors** to API responses

## 📊 Logging Standards

### What to Log
- **All API calls** (external services): Request, response time, cost
- **All errors**: With full context and stack trace
- **Important state changes**: Job status, provider switches
- **Performance metrics**: Render time, frame count, file size
- **Security events**: Auth failures, rate limit hits

### Log Levels
- **debug**: Development details, verbose information
- **log**: Normal operations, state changes
- **warn**: Unexpected but handled situations (fallback triggered)
- **error**: Errors requiring attention
- **fatal**: Critical errors requiring immediate action

## 🧪 Testing Patterns

### Test Structure (AAA Pattern)
```typescript
describe('VideoService', () => {
  let service: VideoService;
  let scriptGenerator: jest.Mocked<IScriptGenerator>;

  beforeEach(async () => {
    const module: TestingModule = await Test.createTestingModule({
      providers: [
        VideoService,
        {
          provide: 'IScriptGenerator',
          useValue: {
            generate: jest.fn(),
          },
        },
      ],
    }).compile();

    service = module.get<VideoService>(VideoService);
    scriptGenerator = module.get('IScriptGenerator');
  });

  describe('generateVideo', () => {
    it('should create video job with valid input', async () => {
      // Arrange
      const dto: CreateVideoDto = { topic: 'Test', format: VideoFormat.MP4 };
      
      // Act
      const result = await service.generateVideo(dto);

      // Assert
      expect(result).toBeDefined();
      expect(result.id).toBeDefined();
    });
  });
});
```

### Testing Rules
1. **One test file per source file** (co-located)
2. **Test behavior, not implementation**
3. **Mock all external dependencies**
4. **Use descriptive test names**: "should {expected behavior} when {condition}"
5. **AAA pattern**: Arrange, Act, Assert (separated by blank lines)
6. **Cover edge cases**: null, undefined, empty arrays, large inputs
7. **Test error paths**: Not just happy paths

## 🎨 DTO & Validation Patterns

### DTO Structure
```typescript
export class CreateVideoDto {
  @ApiProperty({
    description: 'Topic or title for the video',
    example: 'A story about a robot learning to code',
  })
  @IsString()
  @IsNotEmpty()
  topic: string;

  @ApiProperty({
    description: 'Video output format',
    enum: VideoFormat,
    example: VideoFormat.MP4,
  })
  @IsEnum(VideoFormat)
  format: VideoFormat;

  @ApiPropertyOptional({
    description: 'Desired video duration in seconds',
    minimum: 15,
    maximum: 600,
    default: 60,
  })
  @IsOptional()
  @IsInt()
  @Min(15)
  @Max(600)
  duration?: number = 60;
}
```

## 🔐 Security Coding Standards

### Input Validation
- Validate all inputs at API boundaries
- Sanitize user content before processing
- Use class-validator for DTO validation

### Secrets Management
- Always use ConfigService for secrets
- Never hardcode API keys or passwords
- Never log sensitive data

## 🚀 Performance Optimization Guidelines

### Memory Management
- Use object pooling for expensive resources (canvas)
- Always cleanup in finally blocks
- Stream large files, don't load in memory

### Async Optimization
- Use Promise.all for parallel operations
- Avoid sequential await when not needed
- Set timeouts on all external calls
